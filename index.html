Desenvolvimento Frontend

O objetivo deste site é servir como um guia rápido para aprender e revisar alguns conceitos de desenvolvimento web.

Neste guia, vou colocar os materiais que mais gostei, além de algumas notas que acho importantes.

O site roadmap.sh já contém um ótimo guia com passo a passo para aprender desenvolvimento frontend: Frontend Developer

Nota: saber inglês é muito importante, pois muito material bom para estudar programação está em inglês. Por isso, tem muito material em inglês aqui, porém vou linkar também materiais em português para deixar esse guia mais acessível.

HTML

HTML (HyperText Markup Language ou Linguagem de Marcação de Hipertexto) é usado para dar estrutura ao site, que depois pode ser estilizado com CSS e deixar interativo com JavaScript.

Material:

  • MDN: Introdução ao HTML
  • W3Schools: HTML Tutorial

HTML Semântico

Um elemento semântico é um elemento que tem conteúdo bem definido.

Exemplo de elementos semânticos: <p> e <section>.

Exemplo de elementos não-semânticos: <div> e <span>.

Material:

  • web.dev: HTML Semântico
  • W3Schools: HTML Semantics
  • freeCodeCamp(): HTML Best Practices

Escrever HTML semântico ajuda com a legibilidade do código, além de deixar seu site mais acessível.

Tópicos específicos desse assunto:

  • Qual a diferença semântica entre section e article?
  • What's the difference between <b> and <strong>, <i> and <em>?

Acessibilidade

Acessibilidade é a prática de fazer com que os seus sites sejam acessíveis para o maior número de pessoas.

Material:

  • MDN: Boas práticas em acessibilidade
  • w3.org: Developing for Web Accessibility

Dica: a ferramenta Validator feita por W3C ajuda muito para saber se o seu código HTML está bem escrito.

Material:

Position

Usado para definir o tipo de posicionamento de um elemento.

Material:

  • freeCodeCamp(): A propriedade position no CSS explicada e com exemplos

Arquitetura CSS

Modificar o código CSS vai ficando mais difícil conforme o sistema cresce de forma desorganizada (código espaguete).

Usar uma arquitetura no CSS melhora a legibilidade e facilita a manutenção do código.

BEM

BEM é uma metodologia que trabalha com Blocos, Elementos e Modificadores (BEM). É uma convenção popular para dar nome às classes.

BEM é uma aplicação dos conceitos do OOCSS.

Material:

  • BEM: guia definitivo do padrão CSS mais famoso
  • BEM: Introduction
  • BEM: Naming

OOCSS

CSS Orientado a Objetos (OOCSS, sigla em inglês) busca reduzir repetição de código com objetos CSS, que é todo padrão visual que se repetirá no site.

Material:

  • Arquitetura CSS: OOCSS
  • Object Oriented CSS – A teoria das classes reutilizáveis

SMACSS

Arquitetura Escalável e Modular para CSS (SMACSS, sigla em inglês). Seu objetivo principal é categorizar regras CSS.

Material:

  • SMACSS – Scalable and Modular Architecture for CSS
  • SMACSS

Pré-processador

Pré-processador é um programa que permite gerar CSS a partir de uma sintaxe única desse pré-processador, além de adicionar algumas funcionalidades extras que não existem no CSS puro.

Sass

Sass é um pré-processador para CSS que adiciona funcionalidades como variáveis, aninhamento, mixins, entre outros.

Material:

  • Sass Basics
  • Sass: documentation
  • W3Schools: Sass introduction
  • O que é SASS? Venha entender esse novo método de escrever CSS

Nota: @import foi descontinuado e seu uso é desencorajado. Em seu lugar, use o @use e @forward.

Material:

  • Stop using @import with Sass
  • The New SASS Module System: Out with @import, In with @use

Dica: SASS tem integração nativa com BEM.

Dica:Com o SASS, você pode usar a estrutura de arquivos 7-1>: 7 diretórios, 1 arquivo.
